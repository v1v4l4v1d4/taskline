# Docs Tool Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add first-class Markdown documents to tasks so agents stop overwriting task descriptions and can record stage deliverables separately.

**Architecture:** Follow the existing task attachment model. SQLite stores document metadata and the local markdown file path, while the handler owns file IO and injects retrieval URLs. The CLI stays a JSON-over-HTTP wrapper, and the Web task editor adds a Docs section that opens a markdown editor for each document.

**Tech Stack:** Go/Hertz/SQLite server, Cobra CLI, React 19 + TanStack Query + `@uiw/react-md-editor`, existing taskline skill docs.

---

## Product Contract

Tasks get a `docs` array next to `images` and `links`. Each doc has a title, URL, timestamps, and optional markdown content when fetched directly. Docs are stored as `.md` files under `TASKLINE_DOCS_DIR` (local default from `.env.example`: `./.cache/data/docs`), and DB rows only store metadata plus `storage_path`.

Users and agents can create, fetch, update, and delete docs through API and CLI. The task editor shows a Docs section for existing tasks. Clicking a doc opens a full-screen markdown dialog with a title field and markdown editor; adding a doc opens the same dialog empty. The create-task flow shows Docs as unavailable until a task exists, avoiding orphan files.

## API / IDL

```http
POST /api/v1/tasks/:id/docs
Content-Type: application/json

{ "title": "Spec", "content": "# Product design" }
```

```http
GET /api/v1/docs/:id
```

Returns:

```json
{
  "id": "doc-id",
  "task_id": "task-id",
  "title": "Spec",
  "url": "/api/v1/docs/doc-id/content",
  "content": "# Product design",
  "created_at": 1780995001664,
  "updated_at": 1780995001664
}
```

```http
GET /api/v1/docs/:id/content
```

Returns raw `text/markdown; charset=utf-8`.

```http
PATCH /api/v1/docs/:id
Content-Type: application/json

{ "title": "Updated Spec", "content": "# Updated" }
```

```http
DELETE /api/v1/docs/:id
```

Returns `{ "deleted": true, "id": "doc-id" }`.
